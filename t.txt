Summary
Error
Login Failure: all modules ignored — Cognos could not authenticate to Trino via Kerberos JDBC after upgrading IBM JDK 1.8.0_451 → 1.8.0_461 and AIX minor version.

Root Cause
The cron job refreshing the Kerberos credential cache was using the system kinit, which issued a TGT encrypted with DES3-CBC-SHA1. IBM JDK 1.8.0_461 disabled DES3, so when Cognos's JVM tried to read the ccache, it rejected the ticket silently — JAAS reported "all modules ignored".

Solution
Update the cron job to use Cognos's own JDK kinit instead of the system one:

Cognos's JDK kinit defaults to AES256, which IBM JDK 1.8.0_461 accepts. The ccache now contains an AES256 ticket and authentication succeeds.

Key lesson
Always use the same JDK's kinit to populate the ccache as the JVM that will consume it — they must agree on which encryption types are acceptable.
